# Менеджер навчальних предметів (SubjectsManager)

Консольний застосунок для обліку навчальних предметів та пов'язаних із ними занять.

## Предметна область

Застосунок оперує двома основними сутностями:
1. **Предмет (Сутність 1-го рівня):** Характеризується назвою, кількістю ECTS кредитів та сферою знань. 
2. **Заняття (Сутність 2-го рівня):** Належить конкретному предмету. Містить інформацію про дату, час проведення, тему та тип заняття.

## Архітектура рішення

Проєкт розділено на 5 окремих модулів для забезпечення слабкої зв'язності (loose coupling) та легкого масштабування в майбутньому:

1. **`SubjectsManager.CommonComponents`**
   - Бібліотека спільних типів. Містить перечислення (`KnowledgeArea`, `LessonType`), які використовуються на всіх рівнях застосунку.

2. **`SubjectsManager.DBModels`**
   - Класи, що відповідають **виключно за зберігання даних** (`SubjectDBModel`, `LessonDBModel`).
   - Згідно з вимогами, не містять обчислюваних полів та прямих посилань на інші об'єкти (зв'язок реалізовано через `SubjectId`).

3. **`SubjectsManager.UIModels` **
   - Моделі для відображення даних користувачу (`SubjectUIModel`, `LessonUIModel`).
   - Містять обчислювані поля (наприклад, тривалість заняття або загальна тривалість предмета) та колекції дочірніх сутностей (список занять у предметі).

4. **`SubjectsManager.Services` **
   - **`MockDatabase`**: Внутрішнє (`internal`) штучне сховище, попередньо заповнене реалістичними тестовими даними.
   - **`StorageService`**: Сервіс, який відповідає за взаємодію зі сховищем та конвертацію (мапінг) об'єктів бази даних (`DBModel`) у моделі відображення (`UIModel`).

5. **`SubjectsManager.ConsoleApp`**
   - Точка входу в програму. Забезпечує консольний інтерфейс, циклічне меню та введення/виведення даних; делегує запити до `StorageService`.

## Як запустити

1. Відкрийте рішення `SubjectsManager.sln` у Visual Studio.
2. Встановіть проєкт `SubjectsManager.ConsoleApp` як стартовий (Set as Startup Project).
3. Запустіть застосунок (F5).
